    How it actually finds the track

    TIER
    01

    Ask the platform first. It's free, and it's allowed.

    TikTok publishes an oEmbed endpoint that takes a public link and hands back the post's title, author, thumbnail and sound credit. No auth, no key, and it sends CORS headers, so your browser calls it directly with no server at all. That's what this page does, live. When a creator used a licensed track, the credit line is the answer and we stop here. It's the sanctioned embed endpoint, but using it binds you to TikTok's developer terms, there's no published quota, and they can gate it whenever they like.

    Cost $0 Auth none Real right now
    TIER
    02

    When the credit just says "original sound", listen to it.

    This is the case that actually hurts, and the only one worth paying for. It needs a server: pull the audio, fingerprint it, match it against a commercial catalogue. AudD is the realistic option for an indie build at roughly $5 per 1,000 lookups. Apple's ShazamKit is free and has the best catalogue, but it's an SDK you link into an app, not a REST API a web backend can call, and matching against Apple's catalogue ties you to their attribution rules.

    Cost ~$0.005 / lookup Needs 3 to 12 s of audio Typical ~1.4 s
    TIER
    03

    Put it in the playlist. Don't put it in a folder.

    The idea started as "download it". That version doesn't ship: App Store rule 5.2.3 bans saving or converting media from third-party sources without explicit authorization from those sources, and TikTok isn't going to authorize you. Licensed download APIs do exist (7digital sells them), but they're for retailers and they won't sell you the track someone used in a Reel. So Crate does the thing people actually wanted underneath the ask: deep-link to Spotify, Apple Music and YouTube Music, and the Spotify Web API can append straight to a real playlist.

    What works, what doesn't

    The TikTok lookup is real. Paste a TikTok link and this page calls TikTok's own oEmbed endpoint from your browser, live. The video, the creator, the thumbnail and the sound credit you get back are the actual ones. It needs no key, no login and no server, and it costs nothing.

    When the credit names a track, that is the answer and you're done. When it says "original sound", the creator uploaded their own audio and there is no track name to read. That's the case worth paying for, and naming it means fingerprinting the audio on a server. So the page tells you that instead of guessing, because a confident wrong answer is worse than no answer.

    Instagram doesn't work here. Meta's oEmbed refuses browser calls, so Reels genuinely need a backend. And two things worth knowing before building this for real: Shazam already does this natively inside TikTok and Instagram, and fetching the audio for tier 02 is against TikTok's terms. Every existing song-finder does it anyway and survives on obscurity. The honest wedge left is narrow but real: fewer taps, straight to a playlist.
